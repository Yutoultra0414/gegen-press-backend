# 選手データ取込機能について（gegen_press_player_import.html）

## 使い方
1. 管理者アカウントでログインした状態で `gegen_press_player_import.html` を開く
2. 選手名を検索（英語表記推奨。日本語名はWikidataの検索精度上ヒットしないことがあります）
3. 候補一覧から選手を選択
4. 内容を確認・編集（特に「現所属クラブ」はWikidataの自動判定が不完全なため、必ず確認してください）
5. 保存

すでに似た選手が登録されている場合は、新規登録の代わりに「既存選手を更新」を選べます。

## データソース

| 項目 | 取得元 | 備考 |
|---|---|---|
| 選手名・国籍・生年月日・身長・ポジション・所属履歴 | [Wikidata](https://www.wikidata.org/)（検索API + SPARQL Query Service） | CC0。自由に利用可能 |
| 英語版Wikipediaリンク | Wikidata sitelinks | 存在する場合のみ |
| 選手写真 | [Wikimedia Commons](https://commons.wikimedia.org/) | **ライセンス名・作者の両方が確認できた画像のみ**採用。どちらか一方でも取得できない場合は写真を保存せず、デフォルト表示にします |
| 所属クラブ（clubId） | サイト内の `clubs` コレクション（クラブマスタ管理ページ経由） | Wikidataのクラブ名は自動では紐付けず、管理者が手動で選択します |

## 画像について（重要）

- 画像ファイルの実体はコピーせず、Wikimedia Commonsの画像URL（`Special:FilePath`）を直接 `photoUrl` に保存し、`<img src>` で参照しています。Firebase Storageへの保存は行っていません。
- ライセンス（`photoLicense`）・作者（`photoAuthor`）・出典URL（`photoSource`）を選手ドキュメントに保存し、選手プロフィールページの写真下に表示しています（Commonsのライセンスが求める表示義務への対応）。
- Transfermarkt・Getty Images・AFP・Reuters・他ニュースサイト・X（旧Twitter）・Instagramの画像を取得・保存する実装は一切含まれていません。

## 正直に：自動化できていない部分

- **現所属クラブの自動判定はしていません。** Wikidataの「所属履歴」（P54）は現役時代の全クラブを含み、「現在の所属」を正確に判定するのは技術的に信頼できないため、参考情報として表示するのみで、実際のクラブは管理者が選択する設計にしています。
- **利き足（preferredFoot）はWikidataから自動取得していません。** 対応するプロパティの網羅性が低く、正確に取得できる確証が持てなかったため、確認画面での手動入力のみです。
- **背番号はWikidataから自動取得していません。** シーズンごとに変わる情報であり、Wikidata側でも修飾子（qualifier）としての管理でありデータの信頼性・鮮度にばらつきがあるため、手動入力のみとしています。
- **日本語のカタカナ表記ゆれ（例:「エムバペ」）は自動生成していません。** Wikidataの日本語ラベルが登録されている場合はそれを使いますが、無い場合は空欄のまま保存され、確認画面で手動追記が必要です。
